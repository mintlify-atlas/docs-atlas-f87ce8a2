title: "Agent handoff"
description: "How the supervisor delegates tasks to worker agents using tool-based handoffs backed by LangGraph Commands."
---

A **handoff** is the mechanism by which the supervisor transfers control to a worker agent. Handoffs are implemented as ordinary LangGraph tools — the supervisor calls a tool, the tool returns a `Command` that redirects execution to the target agent node, and LangGraph routes the graph accordingly.

## How handoff tools work

For each worker agent, `create_supervisor` automatically generates a handoff tool via `create_handoff_tool`. When the supervisor LLM decides to delegate a task, it calls one of these tools. The tool:

1. Creates a `ToolMessage` confirming the transfer.
2. Constructs and returns a `Command` targeting the worker agent node in the **parent** graph.

The `Command` carries the updated message history as a state update so the worker agent starts with the full context from the supervisor conversation.

```python
# Simplified implementation of create_handoff_tool
from langchain_core.tools import tool, BaseTool, InjectedToolCallId
from langchain_core.messages import ToolMessage
from langgraph.types import Command
from langgraph.prebuilt import InjectedState
from langgraph_supervisor.handoff import METADATA_KEY_HANDOFF_DESTINATION
from typing import Annotated

def create_handoff_tool(
    *,
    agent_name: str,
    name: str | None = None,
    description: str | None = None,
    add_handoff_messages: bool = True,
) -> BaseTool:
    if name is None:
        name = f"transfer_to_{agent_name}"
    if description is None:
        description = f"Ask agent '{agent_name}' for help"

    @tool(name, description=description)
    def handoff_to_agent(
        state: Annotated[dict, InjectedState],
        tool_call_id: Annotated[str, InjectedToolCallId],
    ) -> Command:
        tool_message = ToolMessage(
            content=f"Successfully transferred to {agent_name}",
            name=name,
            tool_call_id=tool_call_id,
            response_metadata={METADATA_KEY_HANDOFF_DESTINATION: agent_name},
        )
        # ... build messages list and return Command
        return Command(
            goto=agent_name,
            graph=Command.PARENT,
            update={**state, "messages": state["messages"] + [tool_message]},
        )

    handoff_to_agent.metadata = {METADATA_KEY_HANDOFF_DESTINATION: agent_name}
    return handoff_to_agent
```

The key points:

- `goto=agent_name` tells LangGraph which node to visit next.
- `graph=Command.PARENT` means the `Command` targets the **outer** supervisor graph, not the internal graph of the supervisor's own ReAct agent.
- The `update` dict is applied to the parent graph's state before the target agent node runs.

## `METADATA_KEY_HANDOFF_DESTINATION`

Every handoff tool has `METADATA_KEY_HANDOFF_DESTINATION` (`"__handoff_destination"`) set in its `.metadata` dict. `create_supervisor` uses this key to:

- Automatically detect which tools are handoff tools vs. regular tools.
- Validate that every worker agent has a corresponding handoff tool when you supply custom tools.
- Extract the target agent name for graph wiring.

```python
from langgraph_supervisor.handoff import METADATA_KEY_HANDOFF_DESTINATION

# Inspect the destination of a handoff tool
tool = create_handoff_tool(agent_name="math_expert")
print(tool.metadata[METADATA_KEY_HANDOFF_DESTINATION])  # "math_expert"
```

The `ToolMessage` returned by the handoff also records the destination in its `response_metadata`:

```python
tool_message.response_metadata  # {"__handoff_destination": "math_expert"}
```

## Single handoffs

When the supervisor calls exactly one handoff tool at a time (the default when `parallel_tool_calls=False`), the flow is straightforward:

```
Supervisor LLM  ──calls──►  transfer_to_math_expert
                                     │
                            Command(goto="math_expert",
                                    graph=Command.PARENT,
                                    update={messages: [..., tool_message]})
                                     │
                                     ▼
                              math_expert node
```

The `Command` carries a direct `goto` string and an `update` dict:

```python
return Command(
    goto=agent_name,          # e.g. "math_expert"
    graph=Command.PARENT,
    update={**state, "messages": handoff_messages},
)
```

## Parallel handoffs

When `parallel_tool_calls=True` and the supervisor LLM calls more than one handoff tool in a single response, each tool invocation returns a `Command` containing a `Send` object instead of a plain `goto` string. LangGraph's `ToolNode` collects all `Send` commands and fans out execution to all target agents simultaneously.

```python
from langgraph.types import Send

# Parallel handoff: each tool returns a Command with Send
return Command(
    graph=Command.PARENT,
    goto=[Send(agent_name, {**state, "messages": handoff_messages})],
)
```

In the parallel case the library also strips out tool calls that are not meant for the current agent from the `AIMessage` before passing the message history to the target agent, ensuring the message history seen by each worker is consistent.

<Tabs>
  <Tab title="Single handoff">
    ```python
    # Default: supervisor calls one agent at a time
    workflow = create_supervisor(
        [research_agent, math_agent],
        model=model,
        parallel_tool_calls=False,  # default
    )
    ```
  </Tab>
  <Tab title="Parallel handoff">
    ```python
    # Supervisor can call multiple agents simultaneously
    workflow = create_supervisor(
        [research_agent, math_agent],
        model=model,
        parallel_tool_calls=True,
    )
    ```
  </Tab>
</Tabs>

<Warning>
  Parallel handoffs require the model to support parallel tool calls (OpenAI and Anthropic). For other providers, control this behavior through the system prompt instead.
</Warning>

## Handoff messages

By default, when a handoff occurs a pair of messages is appended to the shared message history:

- An `AIMessage` from the supervisor containing the tool call.
- A `ToolMessage` confirming the successful transfer.

This pair gives the supervisor visibility into which agents were called and when. You can disable it:

```python
workflow = create_supervisor(
    [research_agent, math_agent],
    model=model,
    add_handoff_messages=False,  # omit the AIMessage + ToolMessage pair
)
```

See [Message history](/concepts/message-history) for a full description of how these messages are structured and how handoff-back messages work when control returns to the supervisor.

## Custom handoff tools

You can replace the automatically generated handoff tools with your own. Pass them via the `tools` parameter. When you do, you must supply a handoff tool for **every** worker agent — `create_supervisor` validates this at construction time.

```python
from langgraph_supervisor import create_handoff_tool

workflow = create_supervisor(
    [research_agent, math_agent],
    tools=[
        create_handoff_tool(
            agent_name="math_expert",
            name="assign_to_math_expert",
            description="Assign a math calculation task to the math expert.",
        ),
        create_handoff_tool(
            agent_name="research_expert",
            name="assign_to_research_expert",
            description="Assign a research task to the research expert.",
        ),
    ],
    model=model,
)
```

You can also customize the prefix used for automatically generated tool names:

```python
workflow = create_supervisor(
    [research_agent, math_agent],
    model=model,
    handoff_tool_prefix="delegate_to_",
)
# Creates tools: delegate_to_math_expert, delegate_to_research_expert
```
